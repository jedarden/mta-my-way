# Mock-Heavy Test Stabilization - bf-134k5

## Summary
Fixed mock cleanup patterns and time-dependent assertions in 7 integration test files to prevent state leaks and ensure consistent test runs.

## Changes Made

### 1. transfer-engine-edge-cases.test.ts
**Issue**: Dynamic re-mocking pattern with module-level variables caused state leaks between tests.
**Fix**: Replaced module-level `mockGetStationsWithBrokenElevators` and `mockGetTravelTimes` variables with proper vi.mock() + vi.mocked() pattern.

```typescript
// Before (problematic):
const mockGetStationsWithBrokenElevators = vi.fn(() => new Set<string>());
const mockGetTravelTimes = vi.fn(() => ({}));
vi.mock("../equipment-poller.js", () => ({
  getStationsWithBrokenElevators: () => mockGetStationsWithBrokenElevators(),
  getTravelTimes: () => mockGetTravelTimes(),
}));

// After (proper):
vi.mock("../equipment-poller.js", () => ({
  getStationsWithBrokenElevators: vi.fn(() => new Set<string>()),
  getTravelTimes: vi.fn(() => ({})),
}));
import { getStationsWithBrokenElevators, getTravelTimes } from "../equipment-poller.js";
// In tests: vi.mocked(getStationsWithBrokenElevators).mockReturnValue(new Set(["725"]));
```

### 2. observability-startup.test.ts
**Issue**: vi.spyOn() calls created mock state that wasn't cleaned up between tests.
**Fix**: Added `vi.clearAllMocks()` in beforeEach and `vi.restoreAllMocks()` in afterEach.

### 3. push-briefing.test.ts, workflows.test.ts
**Issue**: Date.now() calls appeared fragile but were actually using frozen time via vi.useFakeTimers().
**Fix**: Added clarifying comments that Date.now() uses frozen time from vi.useFakeTimers().

### 4. positions-api.test.ts, trip-lookup-api.test.ts, alerts-equipment.test.ts
**Status**: Already had proper beforeEach clearAllMocks + afterEach restoreAllMocks patterns - no changes needed.

## Verification
Ran all 7 test files 3 consecutive times - all passed each run (130 tests total):

## Patterns Established
1. All vi.mock() declarations at module level (top of file)
2. beforeEach: vi.clearAllMocks() + vi.useFakeTimers() + vi.setSystemTime()
3. afterEach: vi.useRealTimers() + vi.restoreAllMocks()
4. Per-test mock setup: vi.mocked(mockedFunction).mockReturnValue(...)
5. Time-dependent assertions: Use frozen time from vi.useFakeTimers(), avoid real Date.now()
