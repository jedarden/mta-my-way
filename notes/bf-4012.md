### MTA API Key Replacement
The current `mta-api-key` is a placeholder: `REPLACE_WITH_REAL_MTA_API_KEY_FROM_HTTPS_API_MTA_INFO`

To complete the setup:
1. Get an MTA GTFS-RT API key from https://api.mta.info/
2. Create a new temporary secret with the real API key
3. Seal it again with kubeseal
4. Replace the current sealedsecret.yaml in declarative-config
5. Commit and push the updated sealed secret

Example command to reseal with real credentials:
```bash
cat > /tmp/mta-my-way-secret-real.yaml <<EOF
apiVersion: v1
kind: Secret
metadata:
  name: mta-my-way-secrets
  namespace: mta-my-way
  labels:
    app.kubernetes.io/name: mta-my-way
type: Opaque
stringData:
  mta-api-key: "YOUR_REAL_MTA_API_KEY_HERE"
  vapid-public-key: "BCPCupCqqaqh9K7kBgHShYUYmbm62-nwqBavwYQuqVMTqejRZeBwJNBNysiWEyM4hoBA1agOnEnJKP0yYnjn5SQ"
  vapid-private-key: "CPOORp9x9t2vkehNK4IvckLoU3l7soMfSZsr8mIDdqA"
EOF

kubeseal \
  --format=yaml \
  --controller-name=sealed-secrets-apexalgo-iad \
  --controller-namespace=sealed-secrets \
  --server=http://traefik-apexalgo-iad:8001 \
  < /tmp/mta-my-way-secret-real.yaml > ~/declarative-config/k8s/apexalgo-iad/mta-my-way/sealedsecret.yaml
```

## Security Notes

1. **VAPID Keys**: These keys are used for web push notifications. The public key is shared with clients, while the private key remains server-side and is now sealed in the cluster.

2. **MTA API Key**: This key is required for accessing MTA GTFS-RT realtime data. It must be obtained from https://api.mta.info/ and should be treated as sensitive.

3. **Sealed Secrets**: The SealedSecret is safe to commit to git. It can only be decrypted by the sealed-secrets controller in the apexalgo-iad cluster.
